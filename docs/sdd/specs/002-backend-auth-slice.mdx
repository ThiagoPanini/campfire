## Goal

Replace the frontend's in-memory mock auth and preferences layer with a real
FastAPI identity backend while keeping scope limited to what the existing
frontend already uses.

## Status

The implementation is present in `apps/api/` and the frontend is wired through
real fetch calls. The source spec still says `Status: Draft`, and several
manual verification tasks remain open in `tasks.md`, so the docs mark this
slice as partial.

## In scope

- User registration and email/password login.
- Opaque access tokens and HttpOnly refresh-token cookies.
- Refresh-token rotation and family revocation on reuse.
- Current-user profile through `GET /me`.
- Full preferences replacement through `PATCH /me/preferences`.
- Development-only Google stub.
- PostgreSQL persistence, Alembic migrations, and OpenAPI snapshot.

## Out of scope

- Real Google OAuth.
- Password reset or password change.
- Repertoire, groups, jam sessions, ratings, notes, and recommendations.
- LocalStack, Terraform, production infrastructure, and CI/CD.
