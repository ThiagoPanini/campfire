<p className="cf-kicker">SPECS · ARTIFACTS</p>

Spec artifacts live beside each spec in `specs/<slice>/`. Keep generated
contracts, data models, research, quickstarts, checklists, and ADRs linked from
the spec landing page.

| Artifact type | Purpose | Current examples |
| --- | --- | --- |
| Contracts | Wire or UI contract for a slice | Spec 001 UI contract, Spec 002 OpenAPI snapshot |
| Data model | Entities, persistence shape, and constraints | Spec 001 mock data model, Spec 002 identity tables |
| Research | Technical choice rationale | Stack and architecture decisions per slice |
| Quickstart | Slice-specific verification path | Frontend MVP and backend auth slice quickstarts |
| Checklists | Requirements quality and acceptance support | `checklists/requirements.md` |
| ADRs | Durable decisions | Spec 002 ADR-0001 through ADR-0006 |
