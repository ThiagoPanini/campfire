---
title: "Debug checklist"
description: "Work through this list before filing an issue."
---

<Steps>
  <Step title="Is the server actually running?">
    ```bash
    curl http://localhost:8000/api/v1/health
    ```
    Expect `{"status":"ok"}`. If not, check uvicorn logs.
  </Step>
  <Step title="Are you on Python 3.11+?">
    ```bash
    python --version
    ```
  </Step>
  <Step title="Is your venv active?">
    Your prompt should start with `(.venv)`. If not, re-activate.
  </Step>
  <Step title="Is the package installed editable?">
    ```bash
    python -c "import campfire; print(campfire.__version__)"
    ```
  </Step>
  <Step title="Did you restart the server since the last test?">
    In-memory storage + regenerated UUIDs means every restart invalidates
    every stored id from the previous run.
  </Step>
  <Step title="Is X-User-Id a seeded id from THIS process?">
    Re-read Alice's id from Swagger after every restart.
  </Step>
  <Step title="Does the payload match the documented shape?">
    Cross-reference [Register repertoire](/features/register-repertoire#request-body).
  </Step>
  <Step title="Check the uvicorn logs">
    The console tab running uvicorn shows every request, the matched route,
    the response status, and any exception traceback. Most bugs are visible there first.
  </Step>
</Steps>

Still stuck? Open an [issue on GitHub](https://github.com/ThiagoPanini/campfire/issues)
and include the failing curl command plus the full uvicorn traceback.
