Every entry below is something people actually hit on a fresh clone.

## Setup

<AccordionGroup>
  <Accordion title="ModuleNotFoundError: No module named 'campfire'">
    The package was not installed in editable mode in the active venv.

    ```bash
    cd backend
    uv sync --extra dev
    # or with pip:
    . .venv/Scripts/activate   # or source .venv/bin/activate
    pip install -e ".[dev]"
    ```
  </Accordion>
  <Accordion title="ModuleNotFoundError: No module named 'fastapi'">
    Same cause. Activate the venv **first**, then install. The prompt should
    show `(.venv)`.
  </Accordion>
  <Accordion title="SyntaxError / ImportError on datetime.UTC or X | Y">
    Python < 3.11. Install 3.11+ and recreate the venv.
  </Accordion>
  <Accordion title="Address already in use / port 8000 busy">
    Another process holds `:8000`. Either stop it or pick a new port:
    ```bash
    uvicorn campfire.main:app --reload --port 8001
    ```
    Then update every client's base URL.
  </Accordion>
  <Accordion title=".env not picked up">
    The file must sit at `backend/.env`, not the repo root, and the name must
    start with a dot. Copy again from `backend/.env.example`.
  </Accordion>
</AccordionGroup>

## Authentication

<AccordionGroup>
  <Accordion title="401 — missing user identity">
    `X-User-Id` header absent. Every route except `/health` needs it.
  </Accordion>
  <Accordion title="401 with a UUID in the detail">
    UUID is well-formed but not one of the seeded users. Re-fetch via Swagger
    after each restart — ids regenerate every boot.
  </Accordion>
</AccordionGroup>

## Routing

<AccordionGroup>
  <Accordion title="404 on /api/v1/healthcheck">
    Typo. The route is `/api/v1/health`.
  </Accordion>
  <Accordion title="404 on /repertoire/possible">
    Group-level "possible repertoire" was removed on purpose. See [Scope](/scope).
  </Accordion>
</AccordionGroup>

## Payload validation

<AccordionGroup>
  <Accordion title="422 on POST /api/v1/repertoire">
    Common triggers:
    - `proficiency` outside 0–10.
    - `proficiency` sent as a string (`"8"`) instead of integer.
    - Empty `song_title`, `song_artist`, or `instrument`.
    - Missing `Content-Type: application/json`.

    Compare your body with [Register repertoire](/features/register-repertoire#request-body).
  </Accordion>
  <Accordion title="422 on search endpoints">
    `limit` outside range:
    - `/songs/search` → 1..100
    - `/instruments` → 1..200
  </Accordion>
</AccordionGroup>

## Business rules

<AccordionGroup>
  <Accordion title="409 Conflict on POST /api/v1/repertoire">
    You already registered the same `(user, song, instrument)`. Change the
    instrument or pick another song.
  </Accordion>
  <Accordion title="Empty [] from /songs/search?q=">
    Empty or whitespace query returns `[]` on purpose. Pass at least one
    non-whitespace character.
  </Accordion>
  <Accordion title="Repertoire is empty after restart">
    Not a bug. In-memory storage resets. See [Scope](/scope).
  </Accordion>
</AccordionGroup>
