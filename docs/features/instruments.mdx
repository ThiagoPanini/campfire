title: "Instruments"
description: "Curated suggestions plus support for custom names at registration time."
---

## Endpoint

```http
GET /api/v1/instruments?query=<text>&limit=<n>
```

- `query` — optional substring filter (case-insensitive).
- `limit` — 1..200, defaults to 50.

Response is a plain list of strings.

## Example

```bash
curl -H "X-User-Id: $USER_ID" \
  "http://localhost:8000/api/v1/instruments?query=guitar"
```

```json
["acoustic guitar", "electric guitar", "bass guitar"]
```

## Custom names are allowed

Suggestions help the future UI populate a dropdown. They do **not** constrain
what you can register. Submitting a `POST /repertoire` with
`"instrument": "mbira"` succeeds even if `mbira` is not in the suggestions.

## Normalization

Instrument names are lowercased and stripped inside the `Instrument` value
object. `"Guitar "`, `"guitar"`, and `"GUITAR"` are all stored as `guitar`.
Two entries that differ only in case collide as the same `(song, instrument)`
and return `409`.
