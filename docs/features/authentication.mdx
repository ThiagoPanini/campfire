---

Campfire ships with `PlaceholderAuthenticator`. Every authenticated route
reads the `X-User-Id` header and looks the UUID up in the in-memory
`UserRepository`.

## What exists today

- Two users seeded at startup: **Alice** and **Bob**.
- IDs regenerate on every process start.
- Every route except `GET /api/v1/health` requires the header.

## The header

```bash
curl -H "X-User-Id: <uuid>" http://localhost:8000/api/v1/users/me
```

| Situation | Status |
|---|---|
| Header absent | `401 Unauthorized` |
| Header present, UUID not seeded | `401 Unauthorized` |
| Header present, UUID seeded | `200` |

## Reading the seeded IDs

Open [Swagger](http://localhost:8000/docs) and execute `GET /api/v1/users` with
any UUID. The `401` body lists both seeded users with their ids. Copy Alice's
id for the walkthroughs.

## Where real auth plugs in

The `Authenticator` Protocol lives in `domain/`. Replace
`PlaceholderAuthenticator` in `backend/src/campfire/infrastructure/auth/` with
your OAuth / magic-link / session adapter. No other layer needs to change.

<Info>
  `PlaceholderAuthenticator.authenticate(user_id)` returns a `User` or raises
  `NotAuthorizedUserError`. Keep that contract; `dependencies.py` maps the
  exception to `401`.
</Info>
