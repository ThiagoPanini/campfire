Campfire is a pre-launch web application for small groups of friends who organize amateur jam sessions: who plays, what they want to play, and when they meet. The codebase is a monorepo with a Vite/React frontend, a Python backend that runs on AWS Lambda, and Terraform modules for AWS-native deployment.

The first implementation slice, `auth-bootstrap`, establishes a secure vertical slice: public entry, managed sign-in, a protected app shell, and a `/me` endpoint that bootstraps a local user record on first login. The active platform work adds a reproducible LocalStack-backed local environment and a Makefile command surface shared by developers and CI.

## Repository layout

| Path | Purpose |
|------|---------|
| `apps/web` | React + Vite frontend with public landing, auth callback, protected shell, and `/me` screen |
| `apps/api` | Python Lambda-style backend with `/health`, `/me`, claims mapping, and local-user bootstrap |
| `apps/shared/contracts` | Implementation-facing API contracts |
| `infra/terraform` | AWS modules for DNS, hosting, Cognito, API Gateway, Lambda, DynamoDB, and observability |
| `specs/001-auth-bootstrap` | Delivered auth-bootstrap feature artifacts |
| `specs/002-cicd-local-env` | Active local environment and CI/CD platform artifacts |
| `scripts/local` | Canonical local workflow scripts used by `make` |
| `docs/guides/platform` | Local environment, CI/CD, and operations documentation |

## Project status

Campfire is under active development. The implemented surface is still foundation-level: authentication, protected entry, local user bootstrap, and local platform workflow. Music-domain features like jam sessions, songs, instruments, and group membership are intentionally future work.
