## Work the auth header

<Steps>
  <Step title="Open GET /api/v1/users">
    Click **Try it out**, leave `X-User-Id` empty, hit **Execute**.
    You will see `401` — that is correct. It confirms the authenticator is active.
  </Step>
  <Step title="Read the seeded ids">
    Scroll down the server logs, or call the same endpoint with a known UUID (any seeded id).
    The response lists both Alice and Bob with their ids.
  </Step>
  <Step title="Paste Alice's id">
    Copy her id into the `X-User-Id` field for every subsequent endpoint.
    Swagger does not persist the header between endpoints for header-based schemes — paste per call.
  </Step>
</Steps>

## Full click-through checklist

| Tag | Endpoint | What to try |
|---|---|---|
| `health` | `GET /api/v1/health` | No header needed. |
| `users` | `GET /api/v1/users` | Copy an id. |
| `users` | `GET /api/v1/users/me` | Echo who you are. |
| `songs` | `GET /api/v1/songs/search?q=beatles` | Four seeded titles. |
| `instruments` | `GET /api/v1/instruments?query=guitar` | Suggestions only. |
| `repertoire` | `POST /api/v1/repertoire` | Prefilled body, set `proficiency:8`. |
| `repertoire` | `GET /api/v1/repertoire/me` | Verify what you just wrote. |

## Common Swagger mistakes

<AccordionGroup>
  <Accordion title="No Execute button visible">
    Click **Try it out** first. Execute only appears once the form is active.
  </Accordion>
  <Accordion title="401 after pasting the UUID">
    Trailing newline or space in the paste. Retype or paste carefully.
  </Accordion>
  <Accordion title="422 with proficiency rejected">
    You sent `"8"` (string) instead of `8` (integer). Remove the quotes.
  </Accordion>
</AccordionGroup>
