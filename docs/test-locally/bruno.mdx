[Bruno](https://www.usebruno.com/) stores every request as a plain-text `.bru`
file, so an API collection is just another folder in the repo.

## Create a local collection

<Steps>
  <Step title="Open Bruno and create a collection">
    Name it `campfire-local`. Point it anywhere under the repo (e.g. `examples/bruno/`).
  </Step>
  <Step title="Create a local environment">
    Variables:
    - `baseUrl` → `http://localhost:8000/api/v1`
    - `userId` → your seeded user UUID

    Activate the `local` environment.
  </Step>
  <Step title="Add a request">
    Example: `GET {{baseUrl}}/songs/search?q=beatles` with header `X-User-Id: {{userId}}`.
  </Step>
</Steps>

## Example `.bru` file

```bru songs_search.bru
meta {
  name: songs_search
  type: http
  seq: 3
}

get {
  url: {{baseUrl}}/songs/search?q=beatles
  body: none
  auth: none
}

headers {
  X-User-Id: {{userId}}
}
```

## Common gotchas

<AccordionGroup>
  <Accordion title="{{baseUrl}} shows literally in the URL">
    You forgot to activate the `local` environment. Click the environment
    selector in the top-right.
  </Accordion>
  <Accordion title="Env changes don't apply">
    Close and reopen the environment edit dialog after saving.
  </Accordion>
</AccordionGroup>
