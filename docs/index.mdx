<Note>
  Campfire is **backend-only** right now. There is no web UI. Everything on this
  site helps you run the API locally and exercise the one slice that exists today.
</Note>

## What Campfire answers today

> **"What songs do I know, on which instruments, and how well?"**

Each authorized user searches songs, picks an instrument, declares a proficiency
score 0–10, and reads their own repertoire back. Group-level questions
("what can we all play tonight?") are intentionally deferred.

## The 60-second tour

<Steps>
  <Step title="Start the backend">
    ```bash
    cd backend
    uv sync --extra dev
    uv run uvicorn campfire.main:app --reload
    ```
  </Step>
  <Step title="List seeded users and copy an ID">
    ```bash
    curl http://localhost:8000/api/v1/users -H "X-User-Id: 00000000-0000-0000-0000-000000000000"
    ```
    The `401` response is expected — open Swagger at [localhost:8000/docs](http://localhost:8000/docs) to read Alice's and Bob's ids.
  </Step>
  <Step title="Register your first entry">
    ```bash
    curl -X POST http://localhost:8000/api/v1/repertoire \
      -H "X-User-Id: <alice-id>" \
      -H "Content-Type: application/json" \
      -d '{"song_title":"Black","song_artist":"Pearl Jam","instrument":"guitar","proficiency":8}'
    ```
  </Step>
  <Step title="Read it back">
    ```bash
    curl http://localhost:8000/api/v1/repertoire/me -H "X-User-Id: <alice-id>"
    ```
  </Step>
</Steps>
