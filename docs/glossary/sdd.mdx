<p className="cf-kicker">GLOSSARY · SPECS</p>

SDD means spec-driven development. Campfire uses specs to define intent,
constraints, acceptance criteria, and verification before implementation.

Why it matters here: docs, code, and generated artifacts should trace back to a
spec for non-trivial work.

Where it appears: [Spec-Kit](/sdd/methodology/spec-kit), [Specs index](/sdd/overview).

Related terms: [Spec-Kit](/glossary/spec-kit), [ADR](/glossary/adr).
