# LocalStack Services

Campfire uses a single LocalStack container as the canonical local AWS plane for both developers and CI integration tests.

## Current contract

- Image: `localstack/localstack:4@sha256:8a2a5b507ae0aae700e4f3ef8527cbbe8af8f14a19a003bd64ed8291a347e33c`
- Endpoint: `http://localhost:4566`
- Region: `us-east-1`
- Volume: `localstack-data`

## Enabled services

- `dynamodb` for the local users table
- `sts` for identity-shaped integration scenarios
- `ssm` for runtime parameters under `/campfire/local/**`
- `secretsmanager` for dummy secret material
- `s3` for local artifact storage flows
- `logs` for log parity with AWS-shaped integrations

## Change control

Any service-set change should update:

1. `docker-compose.backend.yml`
2. `specs/002-cicd-local-env/contracts/localstack-services.md`
3. The seed script in `scripts/local/seed.sh`
4. At least one integration test proving the new service is actually used
