# Local Environment

Campfire's local backend flow uses one root command surface and one AWS-shaped local plane:

- `make` is the human and CI entry point.
- LocalStack provides the shared local AWS endpoint at `http://localhost:4566`.
- The backend runs as a normal Python process so debugger attach stays simple.

## Prerequisites

| Tool | Minimum version | Why |
| --- | --- | --- |
| Docker with Compose v2 | Current stable | Runs LocalStack |
| Python | 3.12 | Matches the backend runtime |
| `uv` | Current stable | Installs backend dependencies |
| Node.js | 20 LTS | Runs frontend tests |
| GNU Make | 4.x | Shared command surface |

Start with:

```bash
make doctor
```

## Core commands

```bash
make doctor
make up
make run
make test
make debug
make down
```

Issue a local JWT for manual API calls:

```bash
TOKEN="$(make -s token)"
curl -H "Authorization: Bearer ${TOKEN}" http://localhost:8010/me
```

## Debugger attach

Use:

```bash
make debug
```

Then attach from:

- VS Code: the committed `.vscode/launch.json` entry named `Python: Remote Attach`
- PyCharm: Python Debug Server on `localhost:5678` with the project source rooted at `apps/api/src`

## LocalStack inspection

```bash
curl http://localhost:4566/_localstack/health
aws --endpoint-url http://localhost:4566 dynamodb list-tables
aws --endpoint-url http://localhost:4566 ssm get-parameters-by-path --path /campfire/local
aws --endpoint-url http://localhost:4566 s3 ls
```

## Offline / restricted network

The following commands work fully offline once dependencies and the LocalStack image are already present locally:

- `make doctor`
- `make down`
- `make run`
- `make debug`
- `make token`
- `make test/backend/unit`

The following commands may need network access on first run:

- `make up` for the first LocalStack image pull
- `uv sync --extra dev`
- `npm ci`

When network access is restricted, `make doctor --check=network` reports a fast failure instead of waiting on a hanging install or image pull.

## Troubleshooting

| Symptom | Likely cause | What to do |
| --- | --- | --- |
| `make up` fails on port `4566` | Another process is already bound there | Stop the conflicting process, then rerun `make up` |
| `make test/backend/integration` fails quickly | LocalStack is not healthy yet | Check `docker compose -f docker-compose.backend.yml logs localstack` and rerun `make up` |
| Debugger does not attach | Wrong port or stale process | Run `make down`, then `make debug`, then attach to `localhost:5678` |
| Seeded resources look stale | Persistent LocalStack volume kept old state | Run `make reset` |
