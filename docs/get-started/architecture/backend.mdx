## Boundaries

The backend is a modular monolith. Each business area is a bounded context under
`apps/api/src/campfire_api/contexts/`. The only implemented context today is
`identity`.

Within a context:

- `domain/` owns entities, value objects, events, and ports.
- `application/` owns use cases and application errors.
- `adapters/` owns FastAPI, SQLAlchemy, Argon2, token generation, rate limiting,
  and clocks.

The architecture rule is dependency direction: business rules do not import
frameworks, ORMs, cloud SDKs, or HTTP concerns.
