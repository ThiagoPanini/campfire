The frontend renders the Campfire MVP screens and preserves the visual system in
`DESIGN.md`. It began as a mock-backed prototype in Spec 001; Spec 002 replaced
the auth boundary with real backend calls, and Spec 004 removed onboarding and
musical preferences from the active app.

## Stack

| Concern | Choice |
| --- | --- |
| Language | TypeScript 5.x |
| Framework | React 19.2 |
| Build / dev server | Vite 8 |
| Routing | In-app routing over a small session store |
| Styling | Plain CSS files (`tokens.css`, `global.css`, `motion.css`) |
| State | In-memory access token and user state; `sessionStorage` for language and accent |
| Testing | Manual acceptance — see [Quickstart](/get-started/introduction/quickstart) |


## Route map

| Route | Screen | Guard |
| --- | --- | --- |
| `/` | Landing | Public |
| `/signin` | Sign In | Public |
| `/signup` | Sign Up | Public |
| `/home` | Home | Session required, else redirect to `/` |
| `/repertoire` | Repertoire | Session required, else redirect to `/` |

Stale `/onboarding` links are not active routes. Signed-out users are sent to
`/`; signed-in users are sent to `/home`.

## Session model

- The access token lives in memory.
- The refresh token is delivered by the backend as an HttpOnly cookie.
- Browser refresh attempts `/auth/refresh` and keeps the user signed in when
  the refresh cookie is valid.
- Language and accent persist for the tab via `sessionStorage`.
- Real OAuth is not implemented. The Google button uses the backend's
  development-only stub endpoint.

## Visual system

The look is defined by [`DESIGN.md`](https://github.com/ThiagoPanini/campfire/blob/main/DESIGN.md):
near-black surfaces, oversized condensed display type, mono uppercase labels,
and one warm orange accent. New screens must reuse the existing tokens and
primitives before introducing new ones.
