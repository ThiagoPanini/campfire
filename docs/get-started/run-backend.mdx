title: "Run the backend"

---

## Start the server

<Tabs>
  <Tab title="uv">
    ```bash
    cd backend
    uv run uvicorn campfire.main:app --reload
    ```
  </Tab>
  <Tab title="pip (venv active)">
    ```bash
    cd backend
    uvicorn campfire.main:app --reload
    ```
  </Tab>
</Tabs>

You should see uvicorn logs ending with:

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

## Confirm it is alive

<CardGroup cols={3}>
  <Card title="Health endpoint" icon="heart-pulse" href="http://localhost:8000/api/v1/health">
    `GET /api/v1/health` → `{"status":"ok", …}`
  </Card>
  <Card title="Swagger UI" icon="book-open" href="http://localhost:8000/docs">
    Interactive endpoint list.
  </Card>
  <Card title="Raw OpenAPI" icon="code" href="http://localhost:8000/openapi.json">
    JSON for code generators and API clients.
  </Card>
</CardGroup>

```bash
curl http://localhost:8000/api/v1/health
```

```json Expected
{"status":"ok","version":"0.1.0"}
```

<Info>
  The bootstrap seeds two authorized users — **Alice** and **Bob** — so the API is
  demoable without real auth. Their UUIDs are generated fresh at every startup.
</Info>

## Change the port

If `:8000` is busy:

```bash
uvicorn campfire.main:app --reload --port 8001
```

Then update the base URL in every curl / Swagger call.
