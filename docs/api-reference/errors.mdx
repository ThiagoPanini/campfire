Campfire returns errors as JSON with a `detail` string:

```json
{"detail": "proficiency score must be between 0 and 10"}
```

## Map

| Status | Meaning in Campfire | Typical cause |
|---:|---|---|
| `200` | OK | Successful GET |
| `201` | Created | Successful `POST /repertoire` |
| `401` | Unauthorized | Missing / unknown `X-User-Id` |
| `404` | Not Found | Wrong route — compare against the endpoint list |
| `409` | Conflict | Duplicate `(user, song, instrument)` on `POST /repertoire` |
| `422` | Unprocessable Entity | pydantic validation failed (empty field, out-of-range proficiency, wrong type) |
| `500` | Internal Server Error | Invariant violation (e.g. `SongNotFoundError` inside `list_user_repertoire`) — means data corruption |
