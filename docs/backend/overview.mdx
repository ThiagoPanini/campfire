---
title: Identity Backend Overview
---

Campfire's first backend slice implements only the identity context: users,
credentials, sessions, refresh tokens, and preferences. Songs, groups, jam
sessions, ratings, notes, recommendations, real Google OAuth, LocalStack, and
Terraform stay deferred to later slices.

The API is a FastAPI modular monolith under `apps/api`. The domain and
application layers are framework-free; FastAPI, SQLAlchemy, Argon2, and token
generation live in adapters. Local development runs the API natively and uses
Compose only for PostgreSQL 16.

## Aurora-readiness checklist

- PostgreSQL-compatible SQL only.
- UUIDs are generated application-side; no Postgres UUID extensions.
- Email normalization and password hashing happen in application code.
- Timestamps are UTC `timestamptz`.
- No session-scoped Postgres features such as temp tables or `LISTEN/NOTIFY`.
- Database URLs flow through `SettingsProvider` so a future secrets provider
  can rotate credentials without changing call sites.
- Migrations contain no `CREATE EXTENSION` statements unless a future slice
  explicitly proves Aurora support.
